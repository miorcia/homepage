Moduł IV

CSS - pozycjonowanie, JS, Serwer, DOM

Display

display: block - powoduje, że element zajmuje całą szerokość i spycha pozostałe elementy niżej. Działa z height i width, oraz max-width i max-height

dispaly: inline - zamienia elementy, które miały domyślny styl przeglądarki display: block, na takie, które zajmują tyle miejsca ile contentu mają w środku. Nie działa na te inline elementy width i height

display: inline-block - układają się inlinowo, ale te, które się nie mieszczą spadają w dół. Działa z width i height

width - ustalona szerokosc na sztywno

height - ustalona wysokosc na sztywno

max-width - wskazuje max szerokosc wyswietlania elementu, ale przy zmniejszeniu okna przeglądarki szerokość będzie się dopasowywać

max-height - wskazuje max wysokosc wisyweitlania elementu, czyli jesli content bedzie dluzszy to wyjdzie poza ten element, bo element nie urosnie

min-width - element ma podane minimum szerokosci i jesli okno przegladarki bedzie mniejsze to sie pojawi scroll

min-height - element ma podane minimum wysokosci, ale jesli content jest dluzszy to sie kontener rozszerzy

overflow: hidden - ucina tresc ktora nie miesci sie w kontenerze

overflow: auto - generuje scrolla contentu w srodku kontenera jesli content sie nie mieści, ale tylko w tych przypadkach jak jest mniejszy

overflow: clip - dziala jak hidden, ale przy bardziej zaawansowanych stronach sie ją stosuje

overflow: scroll - dodaje scrolla dla wszystkich kontenerow, bez znaczenia czy tresc sie miesci, ale nie zadziala jesli content bedzie mniejszy

overflow: visible - tresc wystaje poza kontener

Jednostki

0 to zawsze 0, nie potrzebuje jednostki

px - piksele

% - procenty, ale dla height nie definiujemy, bo przeglądarka często nie wie do czego ma się odnieść, a jesli chcemy użyc procentowych wysokosci to dla html i body musimy ustawic height: 100%

vw - viewport width, szerokosc okna przegladarki, nie bierze pod uwage marignesow domyslnie zdefiniowanych w elementach

vh - viewport height

Odległości

padding-top, padding-right, padding-bottom, padding-left, albo padding: top right bottom left (jak w zegarku) albo padding: top-bottom right-left;

border-width, border-style, border-color albo border: width style color np. border 3px solid #fff;rowniez border-top, border-right itd.

border-radius - działa też bez definicji ramki, piksele lub procenty

margin - tak jak padding z tym, ze jesli dwa elementy wystepuja obok siebie np. box i box-2 i jeden bedzie mial 100px a drugi 120px to one na siebie nachodza i odleglosc miedzy box a box-2 bedzie wynosila 120px, ponnadto margin: top-bottom auto spowoduje ze element zostanie wysrodkowany

box-sizing: content-box (domyslnie, nie trzeba ustawiac) lub border-box - width i height sie bardziej licza, a nie content, czyli jesli ustawimy width: 200px to jaki by nie był padding czy border, box zawsze będzie mial 200px szerokosci, a content sie dostosuje do paddingu i bordera

Pozycjonowanie

position: static - nie jest zdefiniowane, ale bierze pod uwage czy elementy wyswietlaja sie blokowo, jedno pod drugim

position: absolute - ustawia się względem rodzica i względem rodzica możeby go przesuwac używając top, right, botton, left ale krawedz jest liczona od 0x 0y rodzica np. diva ale nie liczymy bordera. Służy np. do przyklejenia czegoś do dołu kontenera i nawet jeśli zwięszymy wielkość kontenera to ten element zostanie przyklejony

z-index: 10 - odpowiada za kolejność warstw. Im wyzsza wartosc z-index ten element będzie wyżej w hierarchii, bo to co jest na koncu dokumentu to sie renderuje pozniej i nadpisuje to co wyzej. Żeby zobaczyć warstwy to w devtoolsach w Miscrosoft Edge można otworzyc ctrl+shift+p i wpisac 3d view

z-index: auto??

position: relative

position: fixed - odpowiada za przyklejenie elementu do okna przeglądarki, np do przyklejenia menu

position: sticky - np. do przyklejenia naglowka gdy mamy wiecej tekstu w kontenerze, ale sticky nie zadziała, gdy mamy overflow. Również nie zadziała np. bez dodania top. Działa tylko w obrębie rodzica - nie wysunie nam się poza rodzica. Zachowuje się blokowo, dopóki nie scrollujemy

float: left - treść opływa dookoła ten element. Nie poleca uzywania tego, ale mozna do obrazka wrzucic, bo nie służy do budowania layoutów, np wyswietlania menu bocznego. Do czyszczenia elementow miedzy dwoma floatami, aby tresc spadnia podnizej tych dwoch sluzy clear: both, ale div z taka klasa tez musi istnieć

Ujednolicanie stylów

Normalizacja - ujednolicanie stylow miedzy przegladarkami, normalize.css - sciagnac link z githuba

Reset - uzycie gwiazdki * {margin:0; outline:none; - usuwa nawet obramowanie focusu, wiec lepiej nie ruszac} powoduje ze wszystkie style są brane pod uwagę, odchodzi do lamusa. Dla innych to moze byc *, *:before, *:hover {}

BEM

BEM = B - block E - element M - modifier, sposob zapisu klas

block__element--modifier

B odpowiada za nadrzędny element

E odpowiada za część bloku (nie musi być bezpośrednim dzieckiem)

M odpowiada za modyfikator - jesli element wystepuje w kilku wariantach tworzymy dodatkowa klase
